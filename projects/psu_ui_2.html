An aftermarket upgrade for a 30-year-old power supply

SHT 1/5

Background

The Keithley 2304A is a decent high-speed benchtop power supply. A two-quadrant supply, the unit can output between 0 and 20 volts, source up to 5 amps, and sink up to 3 amps. It is suitable for testing battery-powered communication devices due to its 40µs recovery time after a large load change. They were also made for years, and can be bought quite cheaply secondhand. The problem? The UI is borderline unusable. The buttons are spongy and hard to press, and the 90s-era LCD can only be viewed from head-on. Time for an upgrade.

SHT 2/5

First Steps

The basic design chosen takes advantage of a built-in feature of the 2304A: the Remote Display Option. Because the 2304A was intended to be mounted in test racks far from your bench, it could be controlled with a remote UI that plugs into the back panel of the unit. Unfortunately, little information about this Remote Display exists, so reverse-engineering of the system is necessary.

DET A — PROTOCOL

Reverse engineering

Through tracing the serial port pins from the original UI microcontroller, through an RS-232 IC, and to the power supply's mainboard, I learned what pins to probe to view the communication traffic.

After that, it was as simple as setting the oscilloscope to Serial one-shot trigger. By recording the values transmitted, I determined that each key press results in a character being sent across the RS-232 bus.

A bit more reverse-engineering revealed that the full text shown on the display (plus a few control characters) is sent after each key press.

SHT 3/5

Embedded Systems and Physical Design

With the communication protocol decoded, the next step is to design a custom embedded system that can speak the same protocol and control the power supply.

It is also necessary to design the user interface and various mechanical components needed to house the electronics. The project requires attention to design to ensure the new UI upgrade is pleasant to operate.

DET B — SCHEMATICS

Schematic capture

The electronics for this project were designed in JLCPCB's EasyEDA software. The physical form factor was prototyped and designed in Fusion 360.

The embedded system design for this project consists of four major components. The first is the CPU, which controls the aftermarket UI upgrade board. The project is based on the ESP32 microcontroller. Because it is a 3.3V device, level shifters interface it with the 5V logic of the display and RS-232 driver ICs.

The power system of the project consists of two DC-DC converters. These step the input 24VDC power rail to the needed voltages, 12V (for fans) and 5V (for ICs, OLED display). The ESP32 is powered by a 3.3V LDO regulator.

The IO sheet contains the RS-232 circuit, OLED display header, and relay drivers for controlling the fan, 4-wire sense, and PSU power relays.

Finally, the UI board contains the buttons used to control the power supply. Kailh Choc V1 mechanical switches were selected to improve the tactile experience of the product.

DET C — MECH / ID

Mechanical and industrial design

The physical design rearranged the buttons from the original power supply to enhance usability, while maintaining access to all features of the original power supply.

It is important to consider the mechanical and usability aspects of any embedded system design. I enjoy creating projects at the intersection between engineering and industrial design to create the best product possible.

SHT 4/5

Board Bring-up and Prototype Assembly

With the design work done, now it is time to assemble and test the prototype. The prototype consists of two PCBAs and associated software. The PCB fab and SMT assembly were contracted to JLCPCB. I installed all through-hole components (keys, display, connectors) and wrote test software to control the system.

DET D — BRING-UP

Board bring-up

The testing and bring-up process for this product was fairly straightforward. Only two modifications were needed to get the system working as designed (accidental use of GPIO 36 and 39 for button input; these pins on the ESP32 don't have internal pull-up resistors).

Because the software application was simple, the Arduino IDE was used to develop the firmware and program the ESP32.

There are individually-addressable RGB LEDs under each key. Here, the "OPERATE" key is highlighted in blue to show the user that the output of the PSU is on.

The last board shown is a simple interconnect PCB that connects the RS-232 output of the main PCB to the RJ-45 jack used by the 2304A.

During testing, the prototype embedded system successfully controlled the Keithley 2304A, exactly achieving the functionality of the OEM Remote Control module.

SHT 5/5

Bonus: Project Case Assembly

I wanted an excuse to play with a CNC router. I enjoy thinking about the mechanical and industrial design of electronics projects, including how the project exists as a finished product. Since a finished product requires a bit more than a pile of PCBs, I need to fabricate a case.

DET E — ENCLOSURE

Case construction

The case was designed in Fusion 360 and routed on a local makerspace's CNC machine. The design language is a bit retro-futuristic, reminiscent of atomic age control panels. I enjoy this style for bench equipment.

The CNC machine used is a variant of the Joe's CNC. The cutting area was 4' by 4', which was overkill for this application. I ended up fabricating 3 cases. Each case has space for 2 UI boards, meaning I have room for future expansion using this panel form factor.

A test-fit with the front panel proved that the aftermarket UI assembly will fit into the CNC-routed case.

The black front panel of the system is actually an aluminum PCB with white silkscreen. The layout of the new controls can be seen in the last image.
