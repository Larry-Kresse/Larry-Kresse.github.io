SHT 1/4

Background

Last year, I built and user tested an off-grid communicator for outdoor situations without cell service. Overall, the tests of the device looked promising. However, the test also identified some areas for improvement.

It is time to design Version 2 of the Celestial Navigator.

SHT 2/4

Hardware Design Changes

The design of this off-grid communicator involved balancing factors across several disciplines of engineering. The 3D printed case needed to be robust and easily manufacturable. The electronic hardware required a custom PCB to connect the myriad of IO devices, in a small and portable form factor. The industrial design should be intuitive for new users, and shouldn't require much explaining or manual-reading.

DET A — ELECTRONICS

Electronics design

The electrical hardware design has much in common with Version 1. The biggest changes include integrating compass, accelerometer, and GPS sensors in the PCB design rather than using external breakout boards. This allows for a thinner, more robust, and more manufacturable design. The soft power switch from Version 1 was also eliminated for simplicity, cost, and ease of use.

The first schematic contains the ESP32 and support circuitry. The second schematic shows the user input components: the switches and knob. Almost nothing about these sheets changed from Version 1.

The third schematic contains the indication LEDs. The main change was the addition of side-firing LEDs for the flashlight. Version 1 had a flashlight, but it wasn't very effective due to using up-firing LEDs with a makeshift reflector.

The fourth schematic sheet contains the sensors and RF components. External breakout boards have been eliminated and all sensors are now integrated on the Navigator's PCB.

The final schematic contains the power management circuitry. The TI TPS61088-based boost converter is still present. A TI BQ29700-based battery protection circuit is implemented to avoid overdischarging the lithium battery. Version 1 had a soft power button and software-controlled battery management, so it could shut itself down if the battery voltage dropped too low. Since Version 2 eliminates the soft power button for simplicity, a hardware solution for battery protection was needed.

DET B — PCB LAYOUT

Board design

The PCB layout is similar to Version 1's layout. Major changes include the 18650 battery mount. V2 has a battery holder that enables hotswapping batteries, where V1's battery was soldered in place. V2 also incorporates a physical power switch, a battery protection circuit, and a soldered-down GPS module.

The only component that is still implemented as a breakout board is the OLED screen, which has doubled in size from V1, measuring 128x64 pixels (MID). There also is an option for a 128x128 pixel OLED (BIG).

Like Version 1, this is a 4-layer board. The top mostly has traces running left-to-right, and the bottom mostly top-to-bottom. This project has been very helpful for leveling up my PCB design skills, and the resulting PCB is much cleaner than Version 1's board.

DET C — MECHANICAL

Mechanical design

V2's physical design shares some commonality with V1. The gray case that fits around the PCBA uses a similar design, and the "shield" pieces printed in an accent color are used again for this project.

Where the design starts diverging is the antenna mount. Where V1 used an unconstrained rubber duck antenna, V2 constrains the antenna in a hinged mounting clip. This is designed to keep the antenna upright using gravity. If the antenna stays upright with respect to the ground, it will perform much better than V1's antenna.

Version 2 also contains a clip over the programming port, allowing access without dismantling the device, and a battery door to enable 18650 battery hot-swapping.

SHT 3/4

Board Bring-up

Board bring-up was extremely fast on this iteration, as it shared much of its design with the already battle-tested Version 1.

SHT 4/4

Prototype Run and User Testing

Once again, the project must be tested with actual users. 30 Celestial Navigator V2 prototypes, consisting of functional PCBAs and 3D printed enclosures, were manufactured.

The V1 Celestial Navigators from last test were also deployed. Prototyping is expensive, it would be a shame to not reuse the old devices.

All 3D printing, through-hole soldering, and final assembly was done in-house.

DET D — FIELD TEST

User testing

The Celestial Navigator V2 continues the pattern of cyberpunk-inspired industrial design. The first picture shows one of the 30 prototypes used in user testing, sending an SOS message. Luckily, nobody in testing needed this feature!

The 30 prototypes were deployed at a camping event. Users were again given a brief crash course on the UI and then released into the wild wielding their communicator. The simplicity of the interface meant that people who used the device last year didn't need a refresher.

The carabiner clip was pushed as the dominant attachment method, as it enables the antenna to be properly positioned. At the event, I noticed that the antenna clip had the desired effect. Antennas of Navigators remained pointed at the sky, the optimal pointing for a monopole rubber duck. Subjective range appeared vastly greater than the V1 test.

The number one problem identified was once again the antenna. Though the carabiner clip worked, the rotating antenna added significant mechanical complexity and failure points. Some of the SMA cables came undone due to vibration. The design was also more difficult to assemble than I'd like.

Version 2 of the Navigator worked even better than Version 1, but I am apparently never satisfied with my projects, so it is time for Version 3 of the Celestial Navigator!
