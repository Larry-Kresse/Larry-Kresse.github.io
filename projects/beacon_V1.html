Celestial Navigator Version 1

An ESP32-based, GPS-enabled 915Mhz communicator for off-grid applications

Background

Camping, hiking, and other outdoor events often take place in areas with weak cell signal. Explorers could benefit from a communication device that uses an independent channel to operate without external infrastructure. The device should enable peer-to-peer communication, wayfinding, and emergency messaging. In addition, users who receive a message should be able to navigate to the sender, allowing people to find eachother in the wilderness.

Initial Design

The design of this off-grid communicator involved balancing factors across several disciplines of engineering. The 3D printed case needed to be robust and easily manufacturable. The electronic hardware required a custom PCB to connect from the myriad of IO devices, in a small and portable form-factor. The industrial design should be intuitive for new users, and should't require much explaining or manual-reading.

Mechanical Design

A clamshell of 3D printed plastic (shown as gray) encased the PCB (the purple form) and provided some internal protection to the PCB and 18650-sized lithium battery (green cylinder).

A "shield" printed in an accent color (red in this case) covered the lithium battery, the GPS antenna, and the power switch. This also helped hold the clamshell together. To save assembly costs, a single screw connects the two halves of the accent color cover together.

The final design is rather rigid and robust. A translucent ring surrounds the screen bezel, and allows LEDs mounted on the PCB in a compass ring to be viewed through the plastic enclosure.

Embedded Systems and Physical Design

With the communication protocol decoded, the next step is to design a custom embedded system that can speak the same protocol and control the power supply.

It is also necessary to design the User Interface and various mechanical components needed to house the electronics.

The project requires attention to design to ensure the new UI upgrade is pleasant to operate.

Schematic capture

The electronics for this project were designed in JLCPCB's EasyEDA software. The physical formfactor was prototyped and designed in Fusion 360.

The embedded system design for this project consists of four major components. The first is the CPU, which controls the aftermarket UI upgrade board. The project is based on the ESP32 microcontroller. Because it is a 3.3V device, level shifters interface it with the 5V logic of the display and RS-232 driver ICs.

The power system of the project consists of two DC-DC converters. These step the input 24VDC power rail to the needed voltages, 12V (for fans) and 5V (for ICs, OLED display). The ESP32 is powered by a 3.3V LDO regulator.

The IO sheet contains the RS-232 circuit, OLED display header, and relay drivers for controlling the fan, 4-wire sense, and PSU power relays.

Finally, the UI board contains the buttons used to control the power supply. The author selected Kalih Choc V1 mechanical switches to improve the tactile experience of the product.

Mechanical and Industrial Design

The physical design rearranged the buttons from the original power supply to enhance usability, while maintaining access to all features of the original power supply.

It is important to consider the mechanical and usability aspects of any embedded system design. I enjoy creating projects at the intersection between engineering and industrial design to create the best product possible.

Board Bring-up and Electronic Prototype Assembly

With the design work done, now it is time to assemble and test the prototype. The prototype consists of two PCBAs and associated software. The PCB fab and SMT assembly were contracted to JLCPCB. I installed all through-hole components (keys, display, connectors) and wrote test software to control the system.

Board Bring-up

The testing and bring-up process for this product was fairly straightforward. Only two modifications were needed to get the system working as designed (Accidental use of GPIO 36 and 39 for button input; these pins on the ESP32 don't have internal pull-up resistors).

Because the software application was simple, the Arduino IDE was used to develop the firmware and program the ESP32.

There are individually-addressable RGB LEDs under each key. Here, the "OPERATE" key is highlighted in blue to show thje user that the output of the PSU is on.

The last board shown is a simple interconnect PCB that connects the RS232 output of the main PCB to the RJ-45 jack used by the 2304A.

During testing, the prototype embedded system sucessfully controlled the Keithley 2304A, exactly achieving the functionality of the OEM Remote Control module.

Bonus content: project case assembly

I wanted an excuse to play with a CNC router. I enjoy thinking about the mechanical and industrial design of electronics projects, including how the project exists as a finished product. Since a finished product requires a bit more than a pile of PCBs, I need to fabricate a case.

Case construction

The case was designed in Fusion 360 and routed on a local makerspace's CNC machine. The design language is a bit retro-futuristic, reminiscent of atmoic age control panels. I enjoy this style for bench equipment.

The CNC machine used is a variant of the Joe's CNC. The cutting area was 4' by 4' which was overkill for this application. I ended up fabricating 3 cases. Each case has space for 2 UI boards, meaning I have room for future expansion using this panel form-factor.

A test-fit with the front panel proved that the aftermarket UI assembly will fit into the CNC-routed case.

The black front panel of the system is actually an aluminum PCB with white silkscreen. The layout of the new controls can be seen in the last image.
